The result is persisted into `capture-state.json` under a new `combined_audio` key. `_print_stop_summary` gains one extra line. Transcription wiring is **not** changed yet — per-source transcripts still appear, plus now a third audio file. End-of-slice: `record stop` returns only after `<timestamp>.wav` is on disk; the user sees it in the folder and in the summary.
